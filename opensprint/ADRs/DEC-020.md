## Question

Given the take-home is single-container scope, what's the right autoscale-related code to include in v1 — and what's the right thing to defer?

## Operator Decision

**Ship the primitives, not the autoscale.** v1 includes everything needed to flip on autoscaling at deploy time without code changes:

- `/healthz` — liveness probe.
- `/readyz` — readiness probe (only true after upstream connect + book initialized).
- `/metrics` — Prometheus exposition with at least `hub_active_consumer_connections`, `hub_active_subscriptions`, `hub_upstream_message_rate`, `hub_upstream_connection_status`.
- `SIGTERM` handler that triggers the drain sequence (DEC-019).
- Rebalance hint message in both WS gateway (`event: "rebalance"`) and MCP (`notifications/rebalance`).

What we do **not** ship: HPA configuration, Terraform, autoscaling policies, the actual deployment.

## Consideration Factors

- **Ship primitives, defer deploy** — turns "we built a monolith that can't scale" into "we built a monolith with all the autoscale primitives in place; flipping on autoscale is an infra change, not a code change." Strong eval signal at minimal scope cost.
- **Ship only what the brief requires** — would skip `/readyz`, /metrics, drain. Loses the "autoscale-ready" demonstration; a reviewer sees a single-container deliverable and nothing more.
- **Ship full autoscale** — includes Terraform, deployment configs, infra. Out of scope for the take-home (DS-BRIEF says single-container deployable, no IaC required).

## Rationale

The eval criterion explicitly weights "scalability and reliability attention." A take-home that has the primitives wired up demonstrates the engineer thought about production operability without overspending the 2-week budget. The marginal cost of these primitives is small (they're standard production hygiene); the signal they send is large.

This is also the kind of intentional scope-cut that reads well in the architecture write-up: *"we made the system autoscale-ready, but did not deploy autoscale, because the brief is single-container and the deferred work is purely operational."*

## Invalidation Trigger

Revisit if:
- The eval feedback indicates the autoscale-readiness reads as scope creep (very unlikely; these are all standard production primitives).
- A specific primitive (e.g., `/metrics`) doesn't fit the chosen libraries cleanly — adapt or drop the specific primitive, but keep the principle.
