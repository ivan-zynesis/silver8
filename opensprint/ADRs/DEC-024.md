## Question

DS-BRIEF says single-container or single-binary deployable, with no requirement for cloud deployment. Should we ship Infrastructure-as-Code, and if so, what target?

## Operator Decision

**No IaC ships in v1.** What ships:

- `Dockerfile` (multi-stage; distroless or alpine final).
- `docker-compose.yml` for local dev (the hub container; optionally a Coinbase WS replay fixture for tests).
- A "Deployment" section in the README sketching the production target *as prose, not code*.

Documented production target (in the architecture write-up):

- **Ingestion tier**: long-running container, *not* function-as-a-service. Cloud Run with `min-instances=1`, or Fly.io machine, or ECS Fargate single task. Runs `MODE=ingestion` once `CoreNetworkModule` exists.
- **Edge/gateway tier**: autoscaled containers — Cloud Run min=1 with autoscale, or Fargate with target-tracking on connection count. Runs `MODE=gateway`.
- **Bus / Registry / OrderBookStore (network)**: Redis pub/sub for Bus, Redis hash/sorted-set for Registry; or NATS JetStream for both with durability. Single managed instance for v1; cluster if needed.
- **Not function-as-a-service**: explicitly rejected for both tiers. Lambda's 15-minute execution cap breaks long-lived upstream WS; per-message Lambda invocation is too expensive and cold-start-prone for hot market data. (Cloudflare Durable Objects is noted as the one genuine serverless answer for this problem shape — one DO per symbol owning upstream + book state — but it's a Cloudflare-specific architecture and not the path we'd take with Node.)

## Consideration Factors

- **Defer IaC, document target** — minimum viable for the brief; lets the architecture write-up name the production shape without spending days on Terraform that the reviewer may never run.
- **Ship one IaC target** (Fly.io is simplest) — half-day cost; signals seriousness. Defensible upgrade if scope allows.
- **Ship multi-cloud IaC** — out of scope for a 2-week deliverable.

## Rationale

The brief is single-container; IaC is implicit-extra-credit at most. A documented production architecture is higher signal per hour than half-finished Terraform. The architecture write-up names the deployment shape so a future operator knows what they're targeting; the deferred work is purely operational.

## Invalidation Trigger

Revisit if:
- Reviewer feedback indicates IaC was expected (would surprise me given the brief, but not impossible).
- Operator pursues actual deployment beyond the take-home; ship a Fly.io or Cloud Run config first.
